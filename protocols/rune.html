Rune

Planned

Visual Byte Glyphs

Vision

Rune is a visual encoding protocol that transforms raw byte data into human-readable glyphs. It is designed as the ultimate analogue fallback‚Äîdata that can be printed on paper and reconstructed by hand if necessary.

Where QR codes are machine-first, Rune is human-first. A system where the encoding itself communicates meaning, not just data.

Principles

  • Every glyph must be reproducible by hand.
  • Encoding must be deterministic and reversible.
  • The glyph set must be culturally neutral and universally readable.
  • Error correction must be visual, not algorithmic.

Rune Roadmap

Phase 1: Glyph Design

Design the core glyph set and define encoding/decoding rules.

Phase 2: Reference Tools

Build encoder/decoder reference tools and visual error correction system.

Integrate Rune encoding with ARC and other Foundation protocols.
